## 🛠️ Technical Architecture

The orchestrator operates on a **three-layer governance model**:

1. **The Observer Layer**  
   Continuously listens to loopback traffic, captures socket statistics, and builds a real-time map of your network's current state.

2. **The Decision Engine**  
   A lightweight, event-driven Bayesian predictor that calculates the optimal number of parallel streams and the ideal chunk size for every active download.

3. **The Actor Layer**  
   Executes the instructions by patching system-level socket buffers and re-routing I/O through a custom virtual network adapter that prioritizes orchestrated traffic.
